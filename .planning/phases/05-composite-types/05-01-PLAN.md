<objective>
Extend BinaryContractSchema.Parse() to handle fixed-count array fields (both scalar and struct element types) with path-based access and enumerable container support.

Purpose: Enables parsing of array fields where the element count is a literal number in the contract. This is the foundation for all composite type support -- semi-dynamic arrays (Plan 02) build on the same element expansion logic.

Output: Parse() handles "array" type nodes in the Pass 1 loop, creates per-element ParsedProperties in an ArrayBuffer, registers path entries in a parse-local NameToOrdinal clone, and returns a ParseResult wired to the ArrayBuffer.
</objective>

  <action>
Modify BinaryContractSchema.Parse(byte[] data) to handle fixed-count array fields. Changes required:

**1. NameToOrdinal clone (CRITICAL -- D-08, Pitfall 1)**
At the start of Parse(), create a parse-local copy of the schema's NameToOrdinal:
```csharp
var parseNameToOrdinal = new Dictionary<string, int>(NameToOrdinal, StringComparer.Ordinal);
```
Use `parseNameToOrdinal` for all element path registrations and pass it to ParseResult instead of `NameToOrdinal`.

**2. ArrayBuffer allocation (D-03)**
Rent an ArrayBuffer at the start of Parse():
```csharp
var arrayBuffer = ArrayBuffer.Rent(initialCapacity: 16, maxOrdinal: 16);
```
Track a sequential `arrayRegionIndex` counter (starts at 0, incremented per array field).

**3. OffsetTable capacity expansion**
Compute expanded capacity at parse time. For fixed arrays, count all element ordinals:
```csharp
int expandedCapacity = TotalOrdinalCapacity;
for (int i = 0; i < OrderedFields.Length; i++)
{
    var node = OrderedFields[i];
    if (node.Type == "array" && node.Count is int fixedCount && node.ArrayElement is not null)
    {
        if (node.ArrayElement.StructFields is not null && node.ArrayElement.StructFields.Length > 0)
            expandedCapacity += fixedCount * node.ArrayElement.StructFields.Length;
        else
            expandedCapacity += fixedCount;
    }
}
```
Use `expandedCapacity` when creating the OffsetTable.

**4. Array handling in the Parse loop (where `fieldType == 0` currently does `continue`)**
Replace the `if (fieldType == 0) continue;` with array handling logic. When `node.Type == "array"`:

a. Read count: `int count = (int)node.Count;` (for fixed arrays, Count is int)
b. Get element info: `var elemInfo = node.ArrayElement;`
c. Graceful degradation (D-05): Clamp count by available payload bytes:
```csharp
int arrayBaseOffset = node.AbsoluteOffset;
int elementSize = elemInfo.Size;
int availableBytes = data.Length - arrayBaseOffset;
int maxFit = elementSize > 0 ? availableBytes / elementSize : 0;
if (maxFit < count) count = maxFit;
```
d. Get the array region index for ArrayBuffer: `int currentArrayRegion = arrayRegionIndex++;`
e. Track a `nextDynamicOrdinal` counter starting at `TotalOrdinalCapacity` for element ordinals.

f. **Scalar array elements (D-02):**
If `elemInfo.Type != "struct"`:
```csharp
byte elemFieldType = GetFieldType(elemInfo.Type);
for (int e = 0; e < count; e++)
{
    int elemOffset = arrayBaseOffset + (e * elementSize);
    string elemPath = node.Name + "/" + e;
    var elemProp = new ParsedProperty(
        data, elemOffset, elementSize, "/" + elemPath,
        /*format:*/ 1, node.ResolvedEndianness, elemFieldType);
    arrayBuffer.Add(currentArrayRegion, elemProp);
    parseNameToOrdinal[elemPath] = nextDynamicOrdinal;
    offsetTable.Set(nextDynamicOrdinal, elemProp);
    nextDynamicOrdinal++;
}
```

g. **Struct array elements (D-07, COMP-03):**
If `elemInfo.Type == "struct"` and `elemInfo.StructFields is not null`:
For each element, iterate struct sub-fields using pre-resolved relative offsets:
```csharp
for (int e = 0; e < count; e++)
{
    int elementBase = arrayBaseOffset + (e * elementSize);
    foreach (var sf in elemInfo.StructFields)
    {
        int sfOffset = elementBase + sf.AbsoluteOffset; // relative offset resolved at load time
        byte sfFieldType = GetFieldType(sf.Type);
        string sfPath = node.Name + "/" + e + "/" + sf.Name;
        var sfProp = new ParsedProperty(
            data, sfOffset, sf.Size, "/" + sfPath,
            /*format:*/ 1, sf.ResolvedEndianness, sfFieldType);
        arrayBuffer.Add(currentArrayRegion, sfProp);
        parseNameToOrdinal[sfPath] = nextDynamicOrdinal;
        offsetTable.Set(nextDynamicOrdinal, sfProp);
        nextDynamicOrdinal++;
    }
}
```
Note: Use `sf.ResolvedEndianness` (not the parent node's) per Pitfall 3.

h. **Container ParsedProperty (D-01, D-03):**
Create the container entry wired to ArrayBuffer for enumeration support:
```csharp
int totalArrayBytes = count * elementSize;
var containerProp = new ParsedProperty(
    data, arrayBaseOffset, totalArrayBytes, "/" + node.Name,
    /*format:*/ 1, node.ResolvedEndianness, FieldTypes.None,
    offsetTable, parseNameToOrdinal, arrayBuffer, currentArrayRegion);
offsetTable.Set(i, containerProp);
```

**5. Update ParseResult construction**
Change the return statement to use the 4-arg constructor with the parse-local NameToOrdinal and ArrayBuffer:
```csharp
return new ParseResult(offsetTable, errors, parseNameToOrdinal, arrayBuffer);
```

**6. Payload length check relaxation**
The existing check `if (TotalFixedSize >= 0 && data.Length < TotalFixedSize)` already handles TotalFixedSize == -1 (dynamic contracts) by skipping the check. No change needed for fixed-only arrays since TotalFixedSize includes them. For semi-dynamic (Plan 02), TotalFixedSize is already -1.

**IMPORTANT:** Do NOT modify the `GetFieldType()` method -- it already returns 0 for "array" and "struct", which is used as the signal to enter the new array handling code. The check should be: when `fieldType == 0`, check `node.Type == "array"` to handle arrays. If `node.Type` is something else unknown, continue as before.

**IMPORTANT:** For struct array elements, the `arrayBuffer.Add()` should add one element PER STRUCT (not per sub-field). Each struct element should be a ParsedProperty with child navigation capability. However, per D-02 and the research, each sub-field gets its own OffsetTable entry and NameToOrdinal path for O(1) lookup. The ArrayBuffer stores either: scalar elements directly, or for struct arrays, a representative element per struct. Looking at the JSON-side pattern in ParsedProperty, the `_arrayBuffer.Get(ordinal, index)` returns a single ParsedProperty per element. For struct arrays, each element in the ArrayBuffer should be a ParsedProperty that represents the struct element (with child navigation pointing to sub-field paths in NameToOrdinal). Actually, the simplest approach consistent with D-02 and D-08: store each sub-field as its own path in NameToOrdinal+OffsetTable for direct O(1) access. For the ArrayBuffer/GetEnumerator pattern, store one entry per element. For scalar arrays, each ArrayBuffer entry IS the element. For struct arrays, each ArrayBuffer entry is a ParsedProperty representing the struct element (one per element index), with _childOrdinals pointing to the parse-local NameToOrdinal so `element["fieldName"]` works. Create the struct element entry:
```csharp
var structElemProp = new ParsedProperty(
    data, elementBase, elementSize, "/" + node.Name + "/" + e,
    /*format:*/ 1, node.ResolvedEndianness, FieldTypes.None,
    offsetTable, parseNameToOrdinal, null, -1);
arrayBuffer.Add(currentArrayRegion, structElemProp);
```
This goes AFTER the sub-field loop for each element. The struct element uses _childOrdinals to resolve `element["code"]` by finding the matching path in NameToOrdinal.
