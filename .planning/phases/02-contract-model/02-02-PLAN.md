    ```csharp
    internal static class BinaryContractValidator
    {
        /// <summary>
        /// Validates a loaded binary contract field dictionary.
        /// Runs validation in phases: (1) types/sizes, (2) graph, (3) type-specific rules.
        /// Collects all errors (does not fail-fast).
        /// </summary>
        internal static bool Validate(
            Dictionary<string, BinaryContractNode> fields,
            ErrorCollector errors)
    }
    ```

    **Phase 2 (types/sizes) — ValidateTypesAndSizes:**
    - Iterate all fields
    - Check each field has Size > 0; if not, add MissingSize error with field name as path
    - Check field.Type is one of the known types: "uint8", "uint16", "uint32", "int8", "int16", "int32", "float32", "float64", "boolean", "string", "enum", "bits", "array", "struct", "padding"

    **Phase 3 (graph) — ValidateGraph:**
    - Count root fields (DependsOn is null). If count != 1, add MissingRoot error. Error message should say "Expected exactly one root field, found {count}."
    - Build reverse map: parentName -> list of children. If any parent has >1 child, add SharedParent error for each extra child. Error message: "Field '{childName}' shares parent '{parentName}' with another field."
    - Validate all DependsOn targets exist in the fields dictionary. If not, add InvalidReference error: "Field '{name}' depends on non-existent field '{dependsOn}'."
    - Cycle detection: for each field, walk DependsOn chain with a HashSet<string> visited set. If revisited, add CyclicDependency error: "Cyclic dependency detected involving field '{name}'."

    **Phase 4 (type-specific) — ValidateTypeSpecificRules:**
    - For each field with Type == "bits":
      - Use bitmask approach: for each sub-field in BitFields, compute mask = ((1u << sub.Bits) - 1) << sub.Bit
      - Check if endBit (sub.Bit + sub.Bits - 1) >= field.Size * 8. If so, add OverlappingBits: "Bit field '{subName}' exceeds container size ({field.Size * 8} bits)."
      - Check if (usedBits & mask) != 0. If so, add OverlappingBits: "Bit field '{subName}' overlaps with another bit field in '{fieldName}'."
      - Accumulate usedBits |= mask

    - For each field with Type == "array" and Count is string (semi-dynamic):
      - Check referenced field exists in dictionary. If not, add InvalidReference: "Array '{fieldName}' count references non-existent field '{countRef}'."
      - Check referenced field type is numeric (uint8/uint16/uint32/int8/int16/int32). If not, add InvalidReference: "Array '{fieldName}' count references non-numeric field '{countRef}'."
      - (Do NOT check ordering here — that's chain resolution's job, and validation runs before resolution)
